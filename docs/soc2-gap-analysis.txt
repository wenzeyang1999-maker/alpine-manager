SOC 2 audits two stages:
  Type I  — Point-in-time snapshot: are controls suitably designed?
  Type II — 6-12 month observation: are controls operating effectively?

The compliance policy (v2.0) is well-written. The primary gaps are in technical
implementation — most controls are documented but not yet in the code.

3. WHAT NEEDS TO BE DONE
--------------------------------------------------------------------------------

  MUST DO (Type I blockers — without these, audit fails):

  [1] Replace demo auth with real authentication
      - Integrate Supabase Auth (email + password, real bcrypt by Supabase)
      - Remove hardcoded demo credentials from login/page.tsx
      - Drop localStorage session, use Supabase JWT sessions
      - Estimated effort: 1-2 weeks dev

  [2] Implement MFA
      - Enable Supabase Auth TOTP (built-in)
      - Require MFA for admin/internal accounts
      - Estimated effort: 2-3 days

  [3] Account lockout after 5 failed attempts
      - Add rate limiting and lockout logic to login API
      - Estimated effort: 1 day

  [4] Audit log table in Supabase
      - Create audit_events table (user_id, action, resource, ip, timestamp)
      - Write to it from all API routes on access/mutation
      - Estimated effort: 3-5 days

  [5] Engage a SOC 2 audit firm
      - Firms friendly to small companies: Johanson Group, Schellman, Prescient
      - They provide a readiness checklist specific to your environment
      - Budget: ~$15,000-30,000 for Type I
      - Timeline: start Q2 2026 to hit Q3 2026 Type I target

  SHOULD DO (complete before Type I audit):

  [6] Error monitoring — Sentry (free tier sufficient)
  [7] Uptime monitoring — BetterUptime or UptimeRobot (free tier sufficient)
  [8] Deployment log — simple Notion page or spreadsheet recording:
      date / commit hash / description / deployed by
  [9] Penetration test — engage external firm (Cobalt, Synack, or freelance)
      Budget: ~$3,000-8,000

  TYPE II ONLY (2027, after 6-12 month observation period):

  [10] Continuous evidence collection — audit logs, access reviews, deploy records
  [11] Annual policy re-acknowledgment workflow for all team members
  [12] Formal backup restoration test with written results
  [13] Annual vendor review documentation
